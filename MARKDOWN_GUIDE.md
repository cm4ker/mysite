# Руководство по использованию Markdown в постах

Теперь вы можете использовать Markdown для форматирования постов! Это позволяет добавлять изображения, заголовки, списки, ссылки и многое другое.

## Основной синтаксис

### Заголовки

```markdown
## Заголовок 2 уровня
### Заголовок 3 уровня
#### Заголовок 4 уровня
```

### Текстовое форматирование

```markdown
**Жирный текст**
*Курсив*
~~Зачеркнутый текст~~
```

### Списки

**Маркированный список:**
```markdown
- Первый пункт
- Второй пункт
- Третий пункт
```

**Нумерованный список:**
```markdown
1. Первый пункт
2. Второй пункт
3. Третий пункт
```

### Ссылки

```markdown
[Текст ссылки](https://example.com)
```

### Цитаты

```markdown
> Это цитата
> Можно писать на нескольких строках
```

### Код

**Инлайн-код:**
```markdown
Используйте `код` внутри текста
```

**Блок кода:**
````markdown
```
function example() {
  console.log("Hello, World!");
}
```
````

## Изображения

### Добавление изображений

1. **Поместите изображение в папку** `public/images/`
   
   Например: `public/images/my-photo.jpg`

2. **Используйте Markdown синтаксис:**
   ```markdown
   ![Описание изображения](/images/my-photo.jpg)
   ```

### Примеры

**Простое изображение:**
```markdown
![Скриншот проекта](/images/screenshot.png)
```

**Изображение с подробным описанием:**
```markdown
![Диаграмма архитектуры системы, показывающая взаимодействие компонентов](/images/architecture.png)
```

### Рекомендации по изображениям

- **Формат:** Используйте JPG для фотографий, PNG для скриншотов и графики
- **Размер:** Оптимизируйте изображения перед загрузкой (рекомендуется до 1MB)
- **Названия файлов:** Используйте понятные имена на английском, без пробелов (например: `project-screenshot.jpg`)
- **Alt-текст:** Всегда указывайте описание в квадратных скобках для доступности

## Полезные советы

1. **Форматирование**: Оставляйте пустую строку между параграфами для правильного отображения
2. **Изображения**: Проверяйте пути к изображениям - они должны начинаться с `/images/`
3. **Предпросмотр**: После добавления поста, проверьте как он отображается на сайте
4. **Резервные копии**: Храните изображения в отдельной папке на случай необходимости восстановления

## Дополнительная информация

Для рендеринга Markdown используется библиотека `react-markdown`. Все стандартные возможности Markdown поддерживаются из коробки.

Если нужны дополнительные возможности (таблицы, сноски и т.д.), можно расширить функциональность через плагины.
